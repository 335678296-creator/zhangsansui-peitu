# 从正文提炼处境（标准模式 + 彩蛋模式）

## 标准模式：5 问提炼法

拿到正文/主题，先回答 5 个问题，再去找物件：

1. **这张图要让谁共鸣？**（互联网打工人、程序员、产品、运营、创作者、一人公司……）
2. **他们正在被什么拉扯？**（被会议拉走、被消息淹、被返工拖、被 AI 替、被筛掉、下班失败……）
3. **这个抽象处境能变成什么物理动作？**（拽、挡、解结、圈改、推、拖、扛、夹）
4. **哪 2-4 个短词能一秒戳中？**（"马上开""谁改的""AI 先做""筛掉""先顶住"）
5. **哪个真实物件天然承载它？**（线缆、纸条、放大镜、筛子、工牌……）

回答完，你就有了：读者 + 处境 + 物理动作 + 真实物件 + 短标签。这就是一张图的全部。

## 优先高共鸣节点

会议、消息、返工、审查、筛选、加码、被替代、下班失败、项目卡住、内容资产沉淀、产品演化断点。这些是最容易一眼"这说的就是我"的处境。

## 彩蛋模式：拆成 5-8 个连续节点

个人经历、项目复盘、产品演化、成长路径，拆成一条线：

- **左侧起点**：身份、起点、最初处境。
- **中间节点**：关键经历、项目转折、内容资产、产品演化断点。
- **右侧终点**：现在在做什么、核心关注、结论或下一阶段。
- 每个节点配：1 个真实物件 + 1 个张三岁动作 + 1-3 行短注释。

节点要形成自然不规律的高低起伏，不是均匀时间轴。

## 事实锚定（强制）

涉及个人经历、品牌名、公司名、项目名、粉丝数、时间跨度、成绩数字时：

- **只能用**用户输入、用户提供素材、或用户明确确认的事实。
- 用户没提供的，**不要**从母版继承，不要补成更完整的履历。
- 无法确认的，用概括标签替代：「内容平台」「项目节点」「用户反馈」「产品实验」。
- 用户要求保留但信息不全 → 在 shot list 或交付说明里标「待确认」，不要画成确定事实。

## 完成标记

- 标准模式：已明确读者处境、核心冲突、真实物件、张三岁动作、短中文标签。
- 彩蛋模式：已得到 5-8 个可视化节点，每个有物件、动作、注释。
- 所有具体事实都能回溯到用户输入；不能回溯的已删除、抽象化或标注待确认。
