# 母版选择（标准模式强制）

标准模式每张图生成前，必须先从 `assets/examples/` 的 `01-06` 里锁定一张最匹配的质量锚点。母版**不是可复制构图**——不要把案例的物件、方位、配件、张三岁姿态直接搬过来。

## 6 张母版 = 6 种动作类型

| 母版 | 动作类型 | 叙事关系 | 适合内容 |
| --- | --- | --- | --- |
| `01-no-topic` 选题枯竭 | 从井里吊起空桶 | "捞不到" | 选题枯竭、素材耗尽、没有灵感 |
| `02-rewrite-pile` 废稿堆 | 被揉皱纸团埋住 | "改到崩" | 改稿、返工、反复推翻、内容重写 |
| `03-flat-metrics` 数据不涨 | 推卡住的大箭头 | "推不动" | 数据停滞、增长卡住、流量不涨 |
| `04-deadline` 赶 DDL | 顶住快漏完的沙漏 | "快来不及" | 截止时间、赶工、交付压力 |
| `05-inspiration` 灵感突现 | 踮脚抓发光灯泡 | "突然有了" | 灵感、转机、想法出现、找到切口 |
| `06-publish-hesitation` 纠结发布 | 手悬在发布按钮上方 | "发不发" | 发布前犹豫、反复检查、担心翻车 |

## 选择原则

- 没选题、没素材、脑袋空 → `01`
- 改稿、推翻、重写、废稿越来越多 → `02`
- 数据不涨、增长卡住、努力推不动 → `03`
- DDL、赶工、快来不及、时间压力 → `04`
- 灵感、转机、突然想到、抓到切口 → `05`
- 发布犹豫、发前检查、纠结要不要按下去 → `06`

内容不完全匹配任何母版时，也要选最接近的一张当**比例/留白/信息密度标尺**，再重写物件、动作和标签。

## 母版锁定必填字段

```text
母版：
抽取的不变量：（这张母版的动作骨架，例如"从物件里努力捞东西却捞空"）
当前内容的变异点（至少 3 个）：
画面 3 秒读懂句：
当前内容适配：
要避免的失败信号：
```

## 变异要求（至少 3 项和母版不同）

主物件类别、空间方向、张三岁动作、道具组合、标签位置、视角或叙事重心——至少 3 项要变。

- 主物件和母版相同 → 空间关系和张三岁动作必须明显不同。
- 空间关系和母版相近 → 主物件、动作、配件必须明显不同。
- **不能把母版做成换字版**。第一眼像旧案例换主题 = 失败。

## 硬性预算

- 只能一个核心物理动作。
- 只能一个真实主物件或紧凑主物件组。
- 最多 1-2 个小配件。
- 最多 4 个短标签，优先 3 个。
- 禁止把主题里所有名词都画出来（元素清单化）→ 先删减再生成。
- 深色大物件不能支配画面；缩小或换更轻的实物锚点。
- 真实物件不要用粉色（和张三岁打架）。
- 整体场景要比母版更轻或同等轻，不能更满更重。
