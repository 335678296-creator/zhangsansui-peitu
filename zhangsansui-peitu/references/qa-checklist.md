# QA 检查清单与失败信号

第一张生成图**永远是候选图，不是交付图**。必须先查看候选图，和所选母版做视觉比对，过了 QA 才交付。

## CRITICAL（不过 = 不交付为最终图）

- [ ] 标准模式有母版锁定记录；彩蛋模式有 4-6 格分镜节点。
- [ ] 标准模式至少 3 个当前内容变异点，不是旧母版换字版。
- [ ] 张三岁出现，并承担**核心物理动作**（不是站旁边）。
- [ ] 张三岁保持**完整形态**，兔耳帽 + 脸的识别特征没丢、没融成一团。
- [ ] 真实物件像同一现场的真实小物，不是截图拼贴，且**压成中性色**（没有粉色物件）。
- [ ] 标准模式 16:9 纯白干净背景；彩蛋模式超横版高级近白背景。
- [ ] 中文标签短、少（≤4，优先 3）、可读，不是假中文或大段解释。
- [ ] 没有元素清单化（把主题名词都堆上）。
- [ ] 没有母版复刻化（物件/方位/配件/姿态直接套用，只换文字）。
- [ ] 画面 3 秒读懂；不靠交付说明就能看懂冲突。
- [ ] 没加用户未提供的人名、联系方式、公司名、Logo、隐私信息。

## STANDARD（不过 = 标注后可继续迭代）

- [ ] 比例、留白、场景覆盖面积对齐母版。
- [ ] 张三岁形体稳定，不像表情包/吉祥物摆拍。
- [ ] 真实物件不压画面，物件之间有空气。
- [ ] 彩色点缀只做节奏（4-6 个小点），张三岁的粉是唯一暖色焦点。
- [ ] 中文没有明显错字（文生图常见问题）。
- [ ] 能说明"哪张最稳、哪张需再收一轮"。

## 失败信号 → 处理

| 失败信号 | 表现 | 处理 |
| --- | --- | --- |
| 元素清单化 | 主题名词全堆上，没有一个清楚物理冲突 | 删到一个主物件+一个动作，重生成 |
| 母版复刻化 | 主体/方向/配件/姿态像旧案例换字 | 换主物件或换空间关系+动作，重生成 |
| 读不懂 | 不看说明看不出冲突 | 强化物理动作（后仰/用力/被压），重生成 |
| 商品图感 | 主物件过大、靠边、居中摆拍 | 缩小物件、让张三岁和它发生接触 |
| 拼贴感 | 物件太多像素材堆 | 删配件、加留白 |
| 标签变说明 | 文字成了概念解释/标题 | 换成 2-6 字情绪短语 |
| 张三岁失败 | 删掉它画面仍完整 | 让它承担核心动作 |
| 中文错字 | 标签笔画错乱 | 减少标注词重生成，或留白事后补字 |
| 背景脏 | 灰白/暖黄/渐变/暗角/贴纸白边 | 提示词强调干净纯白、no sticker border，重生成 |
| 粉色糊 | 物件也是粉、和张三岁混 | 物件改中性色，重生成 |

## 交付状态

- 通过 CRITICAL → 可交付。
- 未过某些 STANDARD → 交付状态标 `DONE_WITH_CONCERNS`，并指出下一轮迭代方向。
- 不合格就重生成或局部编辑，**不要把低质量图当最终交付**。可以告诉用户"候选图未过 QA，正在重做"。
