# 张三岁实物场景图（开源内核版）

## 核心定位

把文章观点、用户处境、个人经历、项目过程，转译成一张可传播的“真实物件小现场”：

```text
张三岁 + 真实物件 + 物理动作 + 短中文标签 + 留白叙事
```

目标：让读者先看到一个真实、轻、可爱的小现场，再在 1 秒内意识到“这说的就是我”。这不是 PPT 信息图，也不是商业插画。

本 Skill 有两种输出规格：

- **标准模式**：16:9 正文配图，一个真实物件小现场，纯白干净背景。
- **彩蛋模式**：张三岁长卷故事图，一条超横版真实物件人生线，用于个人经历、项目复盘、产品演化、成长路径。

## 这是开源内核版

- 角色用**张三岁文字内核**（见 `references/zhangsansui-ip.md`）：5 个核心特征，纯文字即可复现。
- **不依赖参考图、不依赖任何人的 API key**。你用你自己的图像模型/工具（任何支持文生图的都行）。
- 你想要更高保真、更稳定的角色一致性，可以**可选地**把 `assets/zhangsansui-kernel-ref.png` 当参考图喂进支持图生图的模型，但这不是必须的。

## Before Starting

按任务需要读取，不要一次全塞进上下文：

- `references/zhangsansui-ip.md`：张三岁文字内核、动作、形变与配色禁忌。
- `references/style-dna.md`：视觉 DNA、比例、留白、颜色、真实物件规则。
- `references/story-extraction.md`：如何从正文提炼用户处境、物理动作和短标签。
- `references/object-patterns.md`：真实物件选择、场景类型、原创隐喻和反复刻规则。
- `references/master-selection.md`：标准模式 01-06 母版选择、质量锚点、变异要求。
- `references/prompt-template.md`：单张生图提示词模板。
- `references/qa-checklist.md`：生成后质量检查、失败信号和迭代方向。
- `assets/examples/`：高质量母版和出图质量标尺。`01-06` 是标准 16:9 锚点；`07` 是彩蛋长卷母版（如已提供）。对齐它们的比例、留白、物件真实感、张三岁气质、短标签密度、点缀色节奏和动作清晰度；不要复刻物件组合、空间拓扑、张三岁姿态或标签位置。

开始前确认：

- 用户要标准 16:9 正文图，还是彩蛋长卷故事图。
- 用户是否提供了必须保留的人名、品牌名、项目名或经历节点。
- 没明确要求生成图片时，只先输出 shot list 或配图方案。
- 标准模式生图前必须先从 01-06 锁定一张母版；彩蛋模式锁定 07。

## Core Flow

### 1. 消化内容

先提炼“处境”，不要急着找物件：

- 这张图要让谁共鸣？他们正在被什么拉扯、筛选、压住、拽回、催促、重组？
- 抽象概念能变成什么物理动作？
- 哪 2-4 个短词能一秒戳中痛点？
- 哪个真实物件天然承载这个处境？

优先高共鸣节点：会议、消息、返工、审查、筛选、加码、被替代、下班失败、项目卡住。

彩蛋模式先把内容拆成 5-8 个连续节点（左起身份、中段转折、右收当下），每个节点一个真实物件 + 张三岁动作 + 1-3 行短注释。

### 1B. 事实锚定

涉及个人经历、品牌名、公司名、粉丝数、成绩数字时，只能用用户输入或明确确认的事实。无法确认的用概括标签替代（“内容平台”“项目节点”“用户反馈”），不要从母版继承、不要编造履历。

### 2. 列重点 → 问用户要哪些 → 再出 shot list

**默认必做这一步，读完文章不要自作主张直接生图。**

1. 读完文章后，先列出所有【可配图的重点/处境】：**编号 + 每个一句话**（这是什么处境 + 适合画成什么画面）。并告诉用户「这篇我挑出 N 个可配图的重点」。
2. **停下来问用户：这几个里你想给哪几个配图？** 可多选；也允许用户自己补重点或删重点。
3. 用户选定后，只为选中的重点出 shot list，每张写清：用哪张母版、主题、共鸣点、物理动作、真实主物件、张三岁在做什么、中文短标签。
4. **例外**：用户一开始就明确说「直接配 N 张」「你来定」「不用问」时，跳过提问，自己挑最强的 N 个重点直接走。

标准模式 shot list 张数以用户勾选的重点数为准（默认建议 4-8）。

### 2B. 母版锁定（标准模式强制）

读取 `references/master-selection.md`，从 01-06 选一张最匹配的质量锚点。必填：

```text
母版：
抽取的不变量：
当前内容的变异点（至少 3 个）：
画面 3 秒读懂句：
要避免的失败信号：
```

硬性预算：只能一个核心物理动作；只能一个真实主物件或紧凑主物件组；最多 1-2 个小配件；最多 4 个短标签优先 3 个；禁止把主题里所有名词都画出来；深色大物件不能支配画面。

### 3. 单张生成

每张图单独生成。默认 16:9 横版。提示词必须包含（详见 `references/prompt-template.md`）：

- 张三岁 5 个内核特征（团子比例 / 粉兔耳帽 / 棕齐刘海 / 黑豆眼+腮红+小嘴 / 粗描边平涂纯白底）。
- 张三岁承担核心物理动作。
- 真实物件压中性色，统一光影透视。
- 2-4 个短中文手写标签。
- 纯白干净背景 + 足够留白。
- 2-4 个克制的彩色点缀（优先冷色）。
- 负面约束：禁止截图、UI、Logo、PPT、商业插画、贴纸白边、大段解释、粉色物件。

### 3B. 彩蛋长卷生成

用户说“彩蛋模式/长卷/超横版/个人经历/项目复盘/产品演化/成长路径”时使用。锁住结构：超横版 2.6:1~3:1；高级近白背景；一条细黑手绘曲线贯穿；5-8 个真实物件节点沿线自然起伏（非平铺、非正弦波）；每个节点张三岁参与动作；左起右收；节点不编号、文字像手写注释、不放卡片框。

### 4. 使用模板母版

`assets/examples/` 是质量标尺不是描摹模板。对齐它们的：场景覆盖面积、留白、真实物件光影、张三岁与物件比例、短标签数量、点缀色密度、动作类型、叙事关系。当当前内容和某张母版动作类型高度匹配时，用它当质量锚点，抽取动作不变量，再为当前内容写至少 3 个变异点。避免不看内容直接照抄旧案例。

### 5. QA 和迭代

按 `references/qa-checklist.md` 检查。第一张永远是候选图不是交付图。出现以下任一就重写提示词重生成：元素清单化、母版复刻化、读不懂、主物件过大像商品图、物件太多像拼贴、标签变成说明、张三岁没承担核心动作、第一眼不像 `assets/examples/` 的正文配图质量。

### 6. 保存交付

workspace 内工作时，最终图存到 `assets/<topic-slug>-zhangsansui-peitu/`，按序命名 `01-topic.png`。保留原始生成文件，不覆盖已有资产。

## 输出口径

生成前短而准给 shot list；生成后展示图片、说明用途、给保存路径、标出哪张最稳哪张需再收一轮。不要长篇讲理论，让图先说话。

## Quality Gates

### CRITICAL（不过 = 不交付为最终图）

- 标准模式有母版锁定记录；彩蛋模式锁定 07。
- 标准模式至少 3 个当前内容变异点，不是旧母版换字版。
- 张三岁出现并承担核心物理动作；保持完整形态，兔耳帽和脸的识别特征不丢。
- 真实物件像同一现场的真实小物，不是截图拼贴，且压成中性色。
- 标准 16:9 纯白干净背景；彩蛋超横版长卷高级近白背景。
- 中文标签短、少、可读，不是假中文或大段解释。
- 不元素清单化、不母版复刻化、画面 3 秒读懂。
- 不加用户未提供的人名、联系方式、公司名、Logo、隐私信息。

### STANDARD（不过 = 标注后可继续迭代）

- 比例、留白、场景覆盖面积对齐母版。
- 张三岁形体稳定，不像表情包。
- 真实物件不压画面、节点间有空气。
- 彩色点缀只做节奏不抢主体（张三岁的粉是唯一暖色焦点）。
- 能说明“哪张最稳、哪张需再收一轮”。

## Completion States

| 状态 | 含义 |
| --- | --- |
| DONE | 完成 shot list 或图片生成并通过 QA |
| DONE_WITH_CONCERNS | 已交付但有文字稳定性/物件真实感/背景纯度待优化 |
| BLOCKED | 缺主题/正文/节点，或用户要求的事实不可确认 |
